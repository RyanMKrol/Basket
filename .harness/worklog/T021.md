# T021 — 'Usuals' suggestion chips on empty add-bar focus

2026-07-14

## What I did

- `Sources/Services/Suggestions.swift`: added `Suggestions.usuals(history:onList:now:)`
  — a pure, unit-testable ranking that surfaces up to `combinedMax` of the
  user's most-frequent recent items, using the same frequency + recency
  `score` (with an empty query, so only frequency/recency drive it, not
  prefix match), the same 30-day `memoryTTL` cutoff, and the same on-list
  exclusion as `combined`'s history path. `combined` itself is untouched —
  it still returns nothing for an empty query, since it's only ever called
  for the typed-query case now.
- `Sources/Views/ShoppingListView.swift`: `liveSuggestions` now branches —
  when `draft` is empty AND the add bar is focused (`addBarFocused`), it
  calls `Suggestions.usuals(...)` instead of `Suggestions.combined(...)`.
  The result flows through the exact same `AddBar`/`liveSuggestions` wiring
  as typed suggestions, so the chips render and behave identically (tap =
  add) with no new UI code needed.
- Unit tests in `Tests/BasketTests.swift` (`SuggestionsTests`): frequency +
  recency ordering, on-list exclusion, cap at `combinedMax`, empty result
  with no history, and the 30-day forget window.
- New `UITests/UsualsFlowTests.swift`:
  - `testFocusingEmptyAddBarShowsUsualsAndTapAdds` — seeds history by adding
    "Banana", checking it off, and clearing it (so it's remembered but no
    longer on the list), then focuses the empty add bar and asserts the
    "Banana" chip appears (same `addBar.suggestion.<name>` identifier as
    typed suggestions) and tapping it re-adds the item.
  - `testNoUsualsWithoutHistory` — a fresh empty-state install shows no
    chips on focus.
- README: documented the usuals chips under the existing "Quick add" bullet.
