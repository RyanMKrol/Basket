# T027 — "Add to Basket" App Intent + App Shortcut

2026-07-14

## What I did

- Added `Sources/Services/AddItem.swift`: `AddItem.perform(_:context:now:)`, the
  core "add an item" model mutation (case-insensitive dedupe/bump vs. insert +
  `KnownItems.rememberAdd`), extracted out of `ShoppingListView.add` so both the
  add bar and the new Siri entry point behave identically. Refactored
  `ShoppingListView.add` to call it, keeping the view's animation/flash/haptics
  wrapping unchanged.
- Added `Sources/Services/AddToBasketIntent.swift`:
  - `AddToBasketIntent: AppIntent` — takes an `item` parameter, opens the shared
    App Group `ModelContainer` (`AppGroup.storeURL`, from T026) and calls
    `AddItem.perform`, routing "now" through `AppClock.now`.
  - `BasketAppShortcuts: AppShortcutsProvider` — registers the phrases
    "Add \(item) to my Basket" / "Add \(item) to Basket" so Siri/Shortcuts can
    invoke it without opening the app.
  - `BasketItemEntity` + `BasketItemQuery` (`EntityStringQuery`): the current
    AppIntents metadata validator (Xcode 26 SDK,
    `appintentsmetadataprocessor`/`AppIntentsMetadataExtractor`) rejects a plain
    `String` parameter referenced inside a static `AppShortcut` phrase
    ("Invalid parameter type. AppEntity and AppEnum are the only allowed types") —
    confirmed empirically by building with a bare `String` parameter (fails) vs.
    this thin `AppEntity` wrapper whose query echoes back the spoken/typed text
    (`entities(matching:)`), which builds clean and still accepts any free-text
    item name. `AddToBasketIntent.perform` unwraps `item.name` before calling
    `AddItem.perform`.
  - `AddToBasketIntent.containerOverride`: a `TestHooks`-style test seam (nil in
    production) so unit tests can point `perform()` at a scratch in-memory
    container instead of the real App Group store.
- Added `Tests/AddToBasketIntentTests.swift`: drives `AddToBasketIntent.perform()`
  against a scratch in-memory `ModelContainer` via `containerOverride` —
  asserts a new item is inserted with the capitalised name (and that
  `Emoji.forName` derives a real glyph for it), that re-invoking with an
  existing name (different case) bumps the existing row (un-checks it, clears
  `checkedAt`) instead of duplicating, and that `KnownItem` records the add
  either way.
- Updated the README feature list with the Siri/Shortcuts entry.
