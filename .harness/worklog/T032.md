### Implementation
- **`ListLogic.renamed(_:)`** (`Sources/Services/ListLogic.swift`) — the pure
  business rule: trim whitespace, return `nil` to reject an empty result. No
  SwiftData/UIKit dependency, so it's directly unit-testable.
- **`ItemRow`** (`Sources/Views/ItemRow.swift`) — restructured only the
  `showsQuantity == true` (to-get) branch so the **name** is its own tap
  target/accessibility element (`itemRow.name.<name>`, VoiceOver "Rename
  <name>") that swaps to a prefilled `TextField`
  (`itemRow.renameField.<name>`) on tap; submitting calls `onRename` with the
  raw text (empty-rejection happens via `ListLogic.renamed` on the caller
  side). The got-it branch (`showsQuantity == false`) is untouched byte-for-
  byte from before — restore-on-tap-anywhere still behaves exactly as it did.
  Kept the emoji as a queryable-but-otherwise-inert `accessibilityIdentifier`
  + label (`itemRow.emoji.<name>`) so a UI test can assert the re-derived
  glyph without needing `@testable import Basket` (UITests can't see the app
  module's internals).
- **`ToGetSection`** / **`ShoppingListView`** — threaded a new
  `onRename: (GroceryItem, String) -> Void` down to `ItemRow`;
  `ShoppingListView.rename(_:to:)` applies `ListLogic.renamed`, then sets
  `item.name`, clears `quantity`/`unitRaw`, and closes the row's quantity
  editor if it was open. `createdAt` and list position are untouched since
  this mutates the existing item in place (not a delete + re-add). The emoji
  updates for free — `ToGetSection` already recomputes
  `Emoji.forName(item.name)` every render.
- **`A11yIDs.swift`** — added `ItemRow.nameLabel`, `ItemRow.renameField`,
  `ItemRow.emoji`.

    introduces, called out explicitly in the spec).
