# T010: Prefer the head noun (rightmost word) for simple-keyword emoji matches

## 2026-07-12

### Changes Made
- **tools/gen_emoji.py**: Changed the Swift `match` function template emitted
  into `EmojiTable.swift`. When several simple (single-word) keywords match
  different words of a multi-word name, the matcher now prefers the match on
  the RIGHTMOST word of the name (English compounds are head-final), with
  keyword length only breaking ties within the same word. Complex (multi-word)
  keyword matches still take absolute priority over simple matches, unchanged.
  Prepositions ("in"/"of") mark the start of a trailing modifier phrase that
  is excluded from the head-noun search (e.g. "tuna in spring water" — the
  words after "in" are not considered when looking for the head noun, so
  "tuna" wins over "water"); if none of the words before the first such
  stopword match anything, the matcher falls back to the longest match
  anywhere in the full name (the pre-T010 behaviour), so single-word names are
  unaffected.
- **Sources/Services/EmojiTable.swift**: regenerated via
  `python3 tools/gen_emoji.py Sources/Services/EmojiTable.swift` (never
  hand-edited).
- **tools/main.swift**: added a "T010 head-noun (rightmost word) preference"
  check block covering "ginger beer" → 🍺, "carrot cake" → 🎂, "mince pies" →
  🥧, "assam tea" → 🍵, "tuna in spring water" → 🐟, plus regression checks that
  complex-keyword matches ("peanut butter", "sugar snap peas") are unaffected.
- **Tests/BasketTests.swift**: added `testHeadNounPreferredOverModifierWord`
  covering the same cases via XCTest.
