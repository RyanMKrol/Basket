# T039 — Cap the add-bar suggestion stack at 3 rows

## 2026-07-16

### What I did
- Changed `Suggestions.combinedMax` from `4` to `3` in
  `Sources/Services/Suggestions.swift` (line 16). This single constant caps both
  `Suggestions.combined(...)` (live-typing history + dictionary list, via
  `.prefix(combinedMax)`) and `Suggestions.usuals(...)` (empty-query "your usuals"
  list, via `.prefix(combinedMax)`) — the two states of the same add-bar stack.
- Added tests to `Tests/BasketTests.swift`:
  - `testCombinedMaxIsThree` — pins the concrete value (`combinedMax == 3`) so the
    existing symbolic caps tests can't silently drift back to 4.
  - `testCombinedShowsAtMostThreeRows` — 10 matching history items yield exactly 3
    rows from `combined`.
  - `testUsualsShowsAtMostThreeRows` — 10 usuals yield exactly 3 rows from `usuals`.
  - Left `testCapsAtCombinedMax` / `testUsualsCapsAtCombinedMax` untouched (still
    assert against the symbolic `Suggestions.combinedMax`, as the spec requires).
- Did NOT touch `AddBar.swift` / `ShoppingListView.swift` (they consume whatever
  `Suggestions` returns) or any generated file. README uses `combinedMax`
  symbolically ("up to `combinedMax` usuals chips"), so no README change needed.
- Grepped `Tests/` and `UITests/` for any hardcoded literal count of 4 for
  suggestions — none found, matching the spec's note.
