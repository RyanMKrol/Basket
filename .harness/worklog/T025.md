# T025 — Decompose ShoppingListView

2026-07-14

## What I did

Pure refactor of `Sources/Views/ShoppingListView.swift` (533 → 458 lines), extracting
into three new files under `Sources/Views/` and regenerating the Xcode project
(`xcodegen generate`):

- `ToGetSection.swift` — the "to get" `ForEach` row wiring (was inline in `body`).
- `GotItSection.swift` — the faded "Got it" `ForEach` row wiring (was inline in `body`).
- `QuantityController.swift` — the five quantity handlers (`toggle`, `step`, `pickUnit`,
  `setValue`, `clear`), previously private methods on `ShoppingListView`, now a small
  struct holding the `expandedID`/`draft`/`addBarFocused` bindings it needs. Exposed from
  the root view via a `quantity` computed property so call sites (`quantity.toggle(item)`,
  etc.) read the same as before.

No logic changed — bodies were moved verbatim (Haptics calls, `withAppAnimation` usage,
comments, and accessibility identifiers all preserved as-is). `ShoppingListView`'s state,
header, quantity-text/editor factory functions, and the check-off/duplicate-add/purge/
celebration action methods stayed in place, since the task only called for extracting the
section rendering and the quantity handlers.
