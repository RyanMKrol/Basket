# T071 — Widget tap-to-check item intent

**Status:** Done  
**Date:** 2026-07-18

## What was done

Implemented an `AppIntent` (CheckOffItemIntent) that allows tapping item rows in the Home Screen widget (iOS 17+) to check items off without opening the app, moving them to the "Got it" section. The implementation follows the exact pattern of the existing `AddToBasketIntent` (Siri/Shortcuts), with a test seam for hermetic testing.

### Files changed

- **Sources/Services/CheckOffItemIntent.swift** — New app intent that checks off a to-get item by name. Routes through AppSchema for the shared App Group container, sets isChecked=true and checkedAt=AppClock.now (matching in-app check-off), and nudges WidgetReload.reloadTimelines. Gracefully handles already-checked or missing items (idempotent, no-op).
- **Tests/CheckOffItemIntentTests.swift** — 7 unit tests covering:
  - A matching to-get item becomes isChecked==true with checkedAt set
  - Already-checked item is handled gracefully (no crash, no duplicate)
  - Missing item is handled gracefully (no crash, no insert)
  - perform() nudges WidgetReload.reloadTimelines exactly once
  - Widget reload is nudged even when no item is found
  - Item name matching is case-insensitive
  - Whitespace is trimmed from item names
- **BasketWidget/BasketWidgetEntryView.swift** — Wrapped each item row in Button(intent:CheckOffItemIntent) for iOS 17+ (behind #available guard), falling back to static rows on older iOS. Extracted row content into a helper to avoid duplication.
- **project.yml** — Registered CheckOffItemIntent and WidgetReload as sources for the BasketWidgetExtension target so the widget can compile and use the intent.
- **README.md** — Updated the widget feature description to document tap-to-check behavior on iOS 17+ and the fallback for older versions.

## Trade-offs & known limitations

- **Widget intent is iOS 17+ only:** Older iOS versions show static, non-interactive rows. This is acceptable (graceful degradation); intent-based widgets are new in iOS 17.
- **Item targeting by name only:** GroceryItem has no stable UUID (SwiftData's persistentModelID is not reliably reconstructable across processes), so items are matched by name. This is consistent with AddItem's dedup strategy and idempotent (missing/already-checked items are no-op, never inserted).
- **Widget tap-to-check can't be automated tested:** AppIntents only work in the full widget system (device/simulator with WidgetKit running), not in XCTest hosts. Covered by unit tests of the intent's perform() logic and structural compile checks of the Button(intent:) binding.
