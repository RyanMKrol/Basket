# T006 — Return/Enter dismisses the quantity editor's keyboard and commits

2026-07-04

## What I did

In `Sources/Views/QuantityEditor.swift`'s `valueDisplay`, added
`.submitLabel(.done)` and `.onSubmit { fieldFocused = false }` to the editing
`TextField`. Deliberately did not call `commit()` directly from `onSubmit` —
the existing `.onChange(of: fieldFocused)` handler already calls `commit()`
whenever focus leaves, so this reuses that single path instead of double-
invoking `onSetValue`. No other state (`editing`, `ShoppingListView.expandedID`)
is touched by this change.

## Visual/behavioral verification (observed, not just run)

Drove the booted `iPhone 17 Pro` simulator directly with synthetic CGEvent
mouse clicks (`.cgSessionEventTap` — `.cghidEventTap` silently no-ops in this
headless setup) and hardware-keyboard keystrokes, screenshotting between
steps:

1. Tapped "+ Qty" on Tomatoes → row expanded showing stepper + unit pills,
   value pill read "500 g".
2. Tapped the "500 g" value label → it switched into edit mode: on-screen
   decimal keypad appeared, field showed editable "500".
3. Sent hardware-keyboard backspace ×5 then digits "7","5" → field showed
   "75" (on-screen keypad, as expected for `.decimalPad`/`.numberPad`, has no
   built-in Done key — matches the pre-existing comment this task's spec
   references).
4. Sent a hardware-keyboard Return keystroke → on the **next** frame the
   on-screen keypad was still present with the field mid-edit (first Return
   lands right as the hardware-keyboard connection suppresses the software
   keypad — a Simulator quirk, not app behavior). Sent Return a second time
   and confirmed via screenshot: the keypad was gone, the field's row
   (stepper + "−"/"+" buttons and the g/kg/units pills) **remained visibly
   expanded**, and the top-of-row summary pill updated from "500 g" to
   "75 g" — confirming `onSetValue` fired through the existing `onChange(of:
   fieldFocused)` → `commit()` path, and `expandedID` was untouched.

This matches the task's "Done when": Return dismisses the keyboard, commits
the typed value, and leaves the quantity editor row expanded.
