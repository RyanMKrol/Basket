# T006 — Remove non-default themes; hardcode the Pastel Dots look

**Date:** 2026-07-12

## Summary
Basket shipped ~16 themes reachable via `BASKET_THEME`, plus a mutable global
`Theme.current`. Removed all of it — the app now has exactly one look, Pastel
Dots, hardcoded as a constant.

- `Sources/Theme/Theme.swift` — trimmed `FontKind`/`BackgroundStyle` to only
  the cases Pastel Dots uses (`.dotted`); `Theme.current` is now `static let`
  (was `static var`); removed `Theme.all` and `Theme.select(id:)`;
  `BasketBackground` no longer switches over multiple background treatments,
  just renders the dot backdrop directly; removed the per-theme dark-mode
  branch in the time-tint helper (Pastel Dots is always light).
- `Sources/Theme/ThemePresets.swift` — deleted every preset except
  `pastelDots` (soft, pixelPantry, deepDive, cozyCabin, nightArcade, the
  `cozyFresh` factory, and `ThemeStyle.fresh`); `pastelDots` is now a plain
  `ThemeStyle` literal.
- `Sources/BasketApp.swift` — removed the `BASKET_THEME` env-var read and
  `Theme.select(...)` call; `preferredColorScheme` is now a plain `.light`.
- `UITests/AccessibilityAuditTests.swift` — removed
  `testMainListPassesAuditInEveryTheme` (the remaining
  `testMainListPassesAccessibilityAudit` already covers the single look).
- `README.md` — removed the theme / `BASKET_THEME` env var documentation;
  states the look is Pastel Dots plainly; updated the accessibility-audit
  bullet to drop the "each alternate `BASKET_THEME`" mention.
