# T014: Measure.parse rejects negative and malformed input

**2026-07-12**

## What was done

Fixed `Measure.parse` to:
1. Reject inputs starting with "-" (negative values)
2. Parse only the first contiguous number group (digits + optional decimal separator), not all digits across junk ("2x500" → 2, not 2500)
3. Properly handle both "." and "," as decimal separators

Implementation changes in `Sources/Services/Measure.swift`:
- Check for "-" character and reject immediately
- Extract only the first contiguous number by breaking on non-digit/non-decimal characters
- Track `seenDecimal` to prevent multiple decimal points

## Tests added

Added 6 new test cases to `Tests/MeasureTests.swift`:
- `testParsingRejectsNegative()` — "-5" → nil
- `testParsingFirstNumberOnly()` — "2x500" → 2, "750 ml" → 750
- `testParsingDecimalVariants()` — "1.5" → 1.5, "1,5" → 1.5
- `testParsingLeadingDecimal()` — ".5" → 0.5
- `testParsingEmptyInput()` — "" → nil
