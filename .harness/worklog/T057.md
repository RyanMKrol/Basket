# T057: Introduce AppSchema — single source of truth for the SwiftData model set

**Date:** 2026-07-17

## What was done

Added `Sources/Support/AppSchema.swift`:
- `AppSchema.models` — the single `[any PersistentModel.Type]` list (`GroceryItem.self, KnownItem.self`).
- `AppSchema.makeInMemoryContainer()` — `isStoredInMemoryOnly` flavor, for tests/previews.
- `AppSchema.makeContainer(url:)` — explicit file URL flavor, for the UI tests' relaunch/persistence flows.
- `AppSchema.makeSharedContainer()` — `AppGroup.storeURL` flavor, the App Group store the app and the Siri intent (and later a widget) share.

All three factories build a `Schema(models)` once instead of re-spelling the model list, so `AppSchema.models` is the only place `GroceryItem.self`/`KnownItem.self` appear together in `Sources/`.

Adopted the factory at every existing container-creation site:
- `Sources/BasketApp.swift` — all four `init()` branches (`UITEST_STORE_URL`, `-uiTesting`, XCTest-host, production App Group) now call the matching factory. The existing `fatalError`-on-throw wrapper is untouched, per the task's explicit "don't add error recovery" instruction.
- `Sources/Services/AddToBasketIntent.swift` — `resolveContainer()`'s real-store branch now calls `AppSchema.makeSharedContainer()`; the `containerOverride` test seam is unchanged.
- `Sources/Views/ShoppingListView.swift` — the `#Preview`'s `.modelContainer(for:inMemory:)` now passes `AppSchema.models` instead of a hand-spelled array.
- `Tests/ModelTests.swift`, `Tests/AddToBasketIntentTests.swift`, `Tests/RenameLogicTests.swift` — their private `makeContainer()` helpers now delegate to `AppSchema.makeInMemoryContainer()` (still never touching `makeSharedContainer()`, per the task's unit-test rule).

Added `Tests/AppSchemaTests.swift` (new, required by `expectsTest`): asserts that both `makeInMemoryContainer()` and `makeContainer(url:)` (at a `FileManager.default.temporaryDirectory` URL) produce containers whose `schema.entities` name set is exactly `{"GroceryItem", "KnownItem"}` and matches `Schema(AppSchema.models).entities` — so a model added to one flavor but not the other (or a model added to `AppSchema.models` without updating a hardcoded expectation) fails the test.

Updated `README.md`: documented `Sources/Support/AppSchema.swift`'s role near the `AddToBasketIntent` feature description, and added a bullet for `Tests/AppSchemaTests.swift` in the Tests section.
