# T012 — Golden-subset correctness mode for the emoji coverage audit

**Date:** 2026-07-12

## What was done

Extended `tools/audit_coverage.swift` with a CORRECTNESS mode that detects semantic regressions in emoji mappings via a curated golden subset.

### Implementation

1. **Golden subset:** Created a ~184-item golden table spanning 8+ categories:
   - Food staples (grocery): apples, bananas, oranges, carrots, etc.
   - UK/US synonym pairs: coke/pepsi, sprite/fanta, root beer/gatorade, etc.
   - Household: toilet paper, bin bag, clingfilm, paper towel, light bulb, washing powder, etc.
   - Toiletries: shampoo, conditioner, body wash, soap, toothpaste, etc.
   - Pharmacy: aspirin, paracetamol, ibuprofen, calpol, antihistamine, etc.
   - Baby/Pet: pacifier, dummy, baby oil, bib, dog food, cat litter, etc.
   - Brands: kleenex, colgate, andrex, flora, lurpak, philadelphia, nescafe, etc.
   - East Asian: bok choy, daikon, lotus root, bamboo shoots, soy sauce, mirin, etc.
   - South Asian: okra, bitter gourd, curry leaf, turmeric, cumin, etc.
   - Southeast Asian: galangal, lemongrass, thai basil, bird's eye chili, fish sauce, etc.
   - Latam/Caribbean: plantain, yuca, tomatillo, nopal, jalapeño, habanero, cilantro, etc.
   - African: cassava, fufu, garri, yam, peanut, etc.
   - MENA: pita bread, hummus, tahini, falafel, za'atar, sumac, etc.

2. **All expected emoji match the current cascade output** — the golden set captures the current ground truth (curated table + semantic fallback + basket default) so regressions become immediately visible.

3. **Correctness mode entry point:** Pass `-correctness` flag to run the regression test:
   ```sh
   /tmp/audit -correctness
   ```
   - Exits 0 if all golden expectations match; non-zero if any mismatch
   - Prints mismatches clearly: `item → got 🍺, expected 🥤`

4. **Coverage mode preserved:** Regular `audit_coverage.swift` invocation (no flag) still counts corpus coverage by resolution path (curated/semantic/fallback), unchanged from before.

5. **README documentation:** Added two-part audit docs under "Tests" section:
   - Coverage mode: how to count/verify corpus resolve rates
   - Correctness mode: how to run the golden regression test and what categories it covers

## Notes

- The golden set intentionally uses the current cascade output as ground truth, not "ideal" emoji. This captures the exact behaviour we want to regress-test, making it clear when a change accidentally shifts an assignment.
- All 184 items compile and run through the real Emoji.forName cascade, so they exercise the full three-stage pipeline (curated → semantic → fallback).
- The audit runs as a pure Swift program (no simulator needed), making it fast to run during development and in CI.
