# T061 — surface app console errors after launch via xclog

## 2026-07-17

### What was done

Added a best-effort console error check to `build_run.sh` that runs after the app launches and screenshot is captured:

1. **xclog discovery:** The script locates xclog via `XCLOG` environment override first, falling back to the newest Axiom plugin install at `$HOME/.claude/plugins/cache/axiom-marketplace/axiom/*/bin/xclog`.

2. **Graceful degradation:** If xclog is absent or not executable, the script prints a skip note ("xclog not found, skipping console error check") and continues — the check never changes the script's exit code or aborts the run (wrapped with `|| true`).

3. **Error capture:** When xclog is available, it runs `xclog show $APP_NAME --device $SIM --last 2m --filter '(?i)error|fault'` to capture error/fault log lines from the just-launched app over a 2-minute window using xclog's post-mortem show mode (which reads os_log/Logger output already emitted, not relaunching the app).

4. **Output handling:** 
   - If the captured log is non-empty, prints a visible "Console errors detected during launch:" heading followed by the log contents, and keeps the file at `screenshots/console-errors.log` for harness worklog attachment.
   - If empty, deletes the file and prints "No console errors during launch".
   - Either way, exits 0 (this step surfaces problems, it does not gate).

5. **Documentation:** Added two sentences to README.md describing the new behavior (console check via xclog when installed, writes findings to `screenshots/console-errors.log`, skips quietly when xclog is absent).
