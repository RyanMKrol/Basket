# T060: Worklog

## 2026-07-17

### What was done

Implemented two major enhancements to `tools/export_ui_screenshots.sh`:

1. **--failures-only flag**
   - Added optional `--failures-only` flag (accepted anywhere among args)
   - When set, passes `--only-failures` to xcresulttool
   - Exports to `screenshots/ui-tests/failures/` instead of main directory
   - Leaves main directory untouched for partial failure-only exports

2. **Non-fail-fast on test failure**
   - Changed behavior to record xcodebuild exit status in `TEST_FAILED` variable
   - Screenshot export now runs regardless of test success/failure
   - Script exits with proper code at the end (`exit $TEST_FAILED`)

3. **Automatic crash triage sweep**
   - Creates timestamp marker before test run
   - After failed run, sweeps for fresh `.ips` crash reports using the marker
   - Attempts to locate xcsym via:
     - Environment override (`XCSYM=...`)
     - Automatic discovery of newest Axiom plugin version
   - Degrades gracefully:
     - If xcsym found and executable: runs `xcsym crash <file> --format=summary`
     - If xcsym missing/not executable: prints raw .ips paths with note
     - Never fails the script on its own

4. **README.md updates**
   - Added `--failures-only` usage example
   - Added 2-3 sentence explanation of automatic crash triage
   - Documented xcsym dependency and graceful degradation
   - Removed em dash from original text (replaced with "or")
