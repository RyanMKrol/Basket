# T003 — Got-it row body tap no longer unmarks the item

Date: 2026-07-04

## What I did

In `Sources/Views/ItemRow.swift`:

- Body `.onTapGesture` (was line 84): the got-it branch is now a no-op —
  `.onTapGesture { if showsQuantity { onTapQuantity() } }`. To-get rows keep
  opening the quantity editor exactly as before.
- Extracted the emoji/name/quantity-chip cluster into a `@ViewBuilder`
  `rowInfo` computed property so the accessibility setup can differ by
  branch rather than by ternary modifier values. For `showsQuantity == true`
  (to-get rows) it keeps `.isButton`, the "Double tap to set quantity" hint,
  and an `accessibilityAction { onTapQuantity() }` — unchanged from before.
  For `showsQuantity == false` (got-it rows) none of those three are
  attached at all now (previously it had `.isButton`, a "Double tap to
  restore to your list" hint, and an `accessibilityAction { onToggle() }`).
  This was necessary, not just cosmetic: a ternary like
  `.accessibilityAction { showsQuantity ? onTapQuantity() : onToggle() }`
  still attaches the action for got-it rows (as a no-op closure would),
  which fails the "no longer carries an accessibilityAction" requirement —
  the modifier itself must be absent from that branch.
- `CheckCircle`'s own accessibility element (lines ~72-80, label "Got
  it"/"Not got yet", `.isButton`, `accessibilityAction { onToggle() }`) is
  untouched — it's now the sole restore control, for both touch and
  VoiceOver, on both row kinds.
- Updated the file's top doc comment: the "Got it" section note now says
  only the check circle restores the item; tapping the rest of the row does
  nothing.
