# T003 — flash() reads a possibly-deleted SwiftData model after 0.9s

## Build log

**2026-07-12**

### Problem
The `flash(_:)` function in `ShoppingListView.swift` scheduled a delayed closure that accessed `item.persistentModelID` inside the closure. The item could be deleted between scheduling and execution (via the 60-second TTL purge or a "Clear all" operation), creating a window where the app referenced a deleted SwiftData model.

### Solution
Captured the `persistentModelID` as a local variable before scheduling the dispatch, then compared only that captured identifier inside the closure. The closure now holds no reference to the `item` object itself.

**Changed:** `Sources/Views/ShoppingListView.swift` lines 448–455
- Added `let id = item.persistentModelID` before scheduling
- Updated closure to compare `if flashID == id` instead of `if flashID == item.persistentModelID`
