# T009: Require exact-word/plural match for short (<=4 char) emoji keywords

## 2026-07-12

### Changes Made
- **tools/gen_emoji.py**: Changed the `match` function template emitted into
  `EmojiTable.swift` so simple (single-word) keywords of length <= 4 require the
  candidate word to equal the keyword exactly or as a simple plural
  (`word == kw || word == kw + "s" || word == kw + "es"`), instead of the old
  blanket `hasPrefix`. Keywords of length >= 5 keep `hasPrefix`, preserving
  deliberate truncated stems ("strawberr", "anchov", "pastr", etc.).
- Added new curated entries to close corpus coverage gaps that were previously
  (accidentally) covered only via the buggy short-keyword prefix matching:
  `rotini`, `pierogi`, `jamon iberico`, `jambon de bayonne`, `linguica`,
  `rollmops`, `cornichon`, `tarte tatin`, `tealight`, `wrapping paper`,
  `bungeoppang`, `suspiro a la lime`, `sabudana`, `gulab jamun`, `milkfish`,
  `bangus`, `beancurd sheet`, `sago pearl`.
- **Sources/Services/EmojiTable.swift**: regenerated via
  `python3 tools/gen_emoji.py Sources/Services/EmojiTable.swift` (never hand-edited).
- **tools/main.swift**: added a "T009 short-keyword exact-word matching" check
  block covering the false-positive fixes (toilet paper, paper towels, carpet
  cleaner, hamper) and confirming short keywords/plurals ("pap", "ham", "hams",
  "carp", "figs", "peas", "yams") and >=5-char stems ("strawberries") still work.
- **Tests/BasketTests.swift**: added `testShortKeywordsRequireExactWordMatch`
  covering the same cases via XCTest.

### Verification
- Native pure-logic harness (`swiftc ... tools/main.swift`): ALL PASSED, including
  the new T009 block.
- Coverage audit (`tools/audit_coverage.swift` over `tools/corpus/*.txt`):
  0 fall-throughs (confirmed before/after — the fix initially introduced 17 gaps
  from corpus items that were only "covered" via the buggy prefix matching;
  closed by adding proper curated entries, see above).
- Full simulator suite: `xcodegen generate && xcodebuild test -project
  Basket.xcodeproj -scheme Basket -destination 'platform=iOS Simulator,name=iPhone
  17 Pro'` — TEST SUCCEEDED, 40 XCTest cases (including the new
  `testShortKeywordsRequireExactWordMatch`) + 22 UI tests, 0 failures.

### Result
"toilet paper"/"paper towels" no longer match 🥣 via "pap"; "carpet cleaner" no
longer matches 🐠 via "carp"; "hamper" no longer matches 🍖 via "ham"; short
keywords alone and their simple plurals ("pap", "ham"/"hams", "carp", "figs",
"peas", "yams") and >=5-char prefix stems ("strawberries") are unaffected.
Nothing left outstanding for this task.
