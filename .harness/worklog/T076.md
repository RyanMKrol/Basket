# T076: Background blur behind add-bar + suggestions when focused

**Completed:** 2026-07-20 02:11

## What was done

Implemented animated background blur effect for the add bar when focused, with three selectable intensity levels (light/medium/heavy) behind a temporary debug switch.

### Changes made:

1. **Created `Sources/Views/BlurStyle.swift`**: New enum with three blur intensity levels
   - `light`: 3pt blur radius
   - `medium`: 8pt blur radius  
   - `heavy`: 16pt blur radius
   - Each level has a readable label for the selector UI

2. **Modified `Sources/Views/ShoppingListView.swift`**:
   - Added `@State private var selectedBlurStyle: BlurStyle = .medium` to track current blur level
   - Applied `.blur(radius:)` modifier to the main content VStack (header + scrolling item list)
   - Blur radius dynamically switches based on `addBarFocused` state: 0 when unfocused, `selectedBlurStyle.blurRadius` when focused
   - Added `.animation(.easeInOut(duration: 0.2).unlessUITesting, value: addBarFocused)` to animate blur in/out smoothly
   - Inserted temporary debug control: segmented picker above the AddBar that cycles through blur styles
   - Picker only shows when `addBarFocused` is true, allowing live comparison of all three intensities

### Behavior

When the add bar keyboard is open:
- Main content (header + item list) animates to the selected blur level
- AddBar + suggestion rows stay sharp in foreground
- User can tap Light/Medium/Heavy buttons to cycle through blur intensities and see live effect
- Animation uses `withAppAnimation` convention, respecting `.unlessUITesting` so UI tests remain unaffected
- Dismissing keyboard reverses the blur effect smoothly
