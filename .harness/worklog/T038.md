# T038 — Clear-all undo toast: halve duration + stop the Got-it section flickering back on expiry

**Status:** done

## What was done

1. **Shortened toast duration (TestHooks.swift)**: Changed `clearToastDuration` from 5.0s to 2.5s in production (non-UI-testing) mode. Updated the doc comment to reflect the new ~2.5s duration. The UI-testing branch remains at 3.0s (unchanged).

2. **Fixed the Got-it section flicker (ShoppingListView.swift)**: Modified `commitClear()` to narrow the `pendingClearIDs` buffer instead of clearing it entirely. After issuing `context.delete()` calls, the buffer is now filtered to only keep IDs that are still present in `items`: `pendingClearIDs = pendingClearIDs.filter { id in items.contains { $0.persistentModelID == id } }`. This prevents the one-frame race where `pendingClearIDs` was empty but `items` still contained the deleted objects, which caused the "Got it" section to briefly flicker back into view.

3. **Added flicker-verification test (CheckOffFlowTests.swift)**: Added `testClearToastExpiryDoesNotFlickerGotSection()` with `realTiming: true` to exercise the real 2.5s toast duration and SwiftData deletion timing. The test checks an item, taps Clear all, waits out the full ~2.5s expiry without tapping Undo, and confirms the item is gone and the Got it section never reappeared.
