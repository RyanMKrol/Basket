# T017 — Shared accessibility-identifier constants

2026-07-12

## What I did

- Added `Sources/Support/A11yIDs.swift` (`enum A11yID`) declaring every
  accessibility identifier previously duplicated as string literals across
  `Sources/Views/*.swift` and `UITests/*.swift`: `AddBar` (textField,
  addButton, dismissKeyboard, suggestion(name)), `ItemRow` (row(name),
  check(name)), `QuantityEditor` (clear, field, value, increase, decrease,
  unit(symbol)), `Header` (count, aboutButton), `GotSection` (header,
  clearAll). Also added `A11yID.toGetCountText(_:)`, the one shared
  formatter for the header's "N to get" copy — previously built
  independently in `ShoppingListView.swift` and
  `UITests/BasketUITestCase.swift`.
- Wired `Sources/Support/A11yIDs.swift` into the `BasketUITests` target the
  same way `SharedFixtures.swift` already is (`project.yml`, alongside the
  existing `SharedFixtures.swift` source entry), so it compiles into both
  the app and the UI-test bundle from one file.
- Replaced every literal in `Sources/Views/AddBar.swift`,
  `Sources/Views/ItemRow.swift`, `Sources/Views/QuantityEditor.swift`,
  `Sources/Views/ShoppingListView.swift`, and all of `UITests/*.swift`
  (`AccessibilityAuditTests`, `AddItemFlowTests`, `BasketUITestCase`,
  `CheckOffFlowTests`, `PersistenceFlowTests`, `QuantityEditorFlowTests`,
  `SuggestionsFlowTests`, `TapPrecisionTests`) with calls into `A11yID`.
  `BasketUITestCase.waitForToGetCount` and the `gotSectionHeader` accessor
  now use the shared constants/formatter too.
