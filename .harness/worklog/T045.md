## 2026-07-16

### Completed
Implemented the TipJar status state machine as specified, tracking tip purchase feedback states and rendering them in AboutView.

### Changes Made

**Sources/Services/TipJar.swift:**
- Added `TipStatus` enum (Equatable) with cases: idle, purchasing, pendingApproval, failed(message:), thanked
- Added internal `TipOutcome` enum for state machine transitions: success, cancelled, pending, failure(message:)
- Exposed `status` as `private(set) var` observable state
- Implemented state machine core:
  - `beginTip()` — sets .purchasing, clears previous failures on retry
  - `resolve(_ outcome:)` — transitions based on purchase outcome with exact error message
  - `transactionUpdateArrived()` — reconciles pending→thanked when Ask-to-Buy approvals land
- Rewired `tip(_:)` method to call state machine: replaced silent `.pending`/`.userCancelled` with `resolve()`, replaced silent catch with `resolve(.failure(message:))`
- Transaction.updates listener now calls `transactionUpdateArrived()` on verified transactions
- Removed `thanked` Bool in favor of `status == .thanked` for single source of truth

**Sources/Views/AboutView.swift:**
- Replaced `thanked` Bool check with `status == .thanked`
- Added status rendering as quiet line beneath tip buttons:
  - pendingApproval: "Tip pending approval" (soft palette, calm, non-alarming)
  - failed: displays exact message "Couldn't reach the App Store. Please try again later."
  - idle/purchasing: shows "Thanks for supporting Basket ♥" if previously tipped
- Added `accessibilityIdentifier("about.tipStatus")` to status line for testing
- Fixed em dash in accessibility hint: changed "Leaves a tip — doesn't unlock anything" to "Leaves a tip. Doesn't unlock anything."

**Tests/TipJarTests.swift:**
- Added comprehensive unit tests for state machine (12 new tests), all passing:
  - Initial status is idle
  - beginTip sets purchasing
  - pending → pendingApproval
  - failure → failed with exact message
  - cancelled → idle
  - success → thanked (with 3s revert to idle)
  - failed clears when new tip begins (retry)
  - transactionUpdateArrived: pendingApproval → thanked
  - transactionUpdateArrived: other states mark tipped without state change
