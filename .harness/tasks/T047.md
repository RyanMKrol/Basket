## Overview
The loop's local Definition-of-Done gate (.harness/config/harness.env LOCAL_DOD) hand-lists the native-harness swiftc invocation over Sources/Services/*.swift + tools/main.swift. The follow-up BasketCore tasks MOVE those files into a local SwiftPM package and later delete tools/main.swift entirely, so the current LOCAL_DOD would fail every subsequent loop attempt the moment the extraction lands. harness.env is harness config the loop must never edit itself (scope gate) — this is a one-time OWNER edit, and it must land BEFORE the extraction task runs.

## Do

Owner (human) edits `.harness/config/harness.env` line 29, replacing the LOCAL_DOD default with a form that is green before, during, and after the extraction:

```sh
: "${LOCAL_DOD:=xcodegen generate && xcodebuild test -project Basket.xcodeproj -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro' && { [ ! -d BasketCore ] || swift test --package-path BasketCore; }}"
```

Why this shape: before the package exists the `[ ! -d BasketCore ]` guard skips the swift-test leg (the simulator suite still covers all the same pure logic via Tests/, and CI's native-harness step keeps running until it is retired); once BasketCore lands, `swift test --package-path BasketCore` becomes the fast native leg automatically. One edit, no second touch needed after the retirement task.

Commit directly to main (harness config is meant to be owner-edited).
