## Overview

The "All done!" celebration — the app's emotional peak — is silent; add a success haptic, and one
on restore.

## Do

In the check-off flow in `Sources/Views/ShoppingListView.swift` /
`Sources/Services/Haptics.swift`: fire a `.success` notification haptic when the "All done!"
celebration triggers (the moment the last to-get item commits, alongside the existing celebration
overlay), and a light impact haptic when restoring an item from the Got-it section (currently
silent). Route through the existing `Haptics` service; while in there, hold static prepared
generators (`prepare()`) instead of allocating per call (the review's Haptics nit) — behaviour
otherwise unchanged. Haptics must not fire under UI testing if the existing service already
suppresses them there — follow the current pattern.

## Done when

- `Haptics` exposes the success + restore triggers, uses stored prepared generators, and both call
  sites fire at the right moments (celebration trigger, got-row restore).
- No UI-test flakiness introduced: full simulator suite green
  (`xcodegen generate && xcodebuild test -project Basket.xcodeproj -scheme Basket -destination
  'platform=iOS Simulator,name=iPhone 17 Pro'`).
- README's "little touches" list mentions the celebration haptic.
