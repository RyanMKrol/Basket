## Overview
The human half of the T069 split. review-t069-a lands the script + docs but deliberately cannot produce a real baseline, because on the loop's build machine `xctrace --attach` never sees the live Simulator-hosted Basket process (empty trace) and `--device` misroutes to a paired iPhone and hangs. Capturing a genuine, non-empty CPU trace with Basket-attributed hot frames therefore requires a human running `tools/profile_app.sh --refresh-baseline` on hardware where attach works (or via the Instruments GUI, exporting an equivalent analyze JSON) and committing the result. This is a needs-human task precisely because no unattended cold pass can satisfy it — that is exactly why the two prior automated attempts blocked.
