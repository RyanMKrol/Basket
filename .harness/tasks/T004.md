## Overview
In ShoppingListView's titleView, the heart.fill icon shown next to "Basket" once the user has tipped looks vertically off relative to the title text. The title is rendered in the app's default pixel/bitmap font (Theme.title, which for the shipped Pastel Dots theme resolves to .custom("Silkscreen", scale: 0.78)), and the HStack currently aligns children on .firstTextBaseline, which is unreliable when pairing a bitmap font (with atypical baseline/ascender metrics) against a system SF Symbol.

## Do
In Sources/Views/ShoppingListView.swift, in the `titleView` computed property (~lines 161-173), change the HStack's alignment from `HStack(alignment: .firstTextBaseline, spacing: 7)` to `HStack(alignment: .center, spacing: 7)` so the heart.fill icon is centered against the title's layout bounding box rather than a baseline that the custom pixel font reports inconsistently. After making that change, build and run the app (./build_run.sh, which saves a screenshot to screenshots/) with a tip already recorded (or temporarily force `supporter` truthy while checking) and visually inspect the header: if the heart still looks noticeably off-center relative to the vertical midline of "Basket"/the rainbow title, nudge it into place with a small manual vertical adjustment (e.g. `.offset(y: ...)` or `.alignmentGuide(.center)` on the Image only, not the Text), choosing whatever small offset value makes it look centered in the screenshot. Do not change the outer `header` HStack (~line 186) or any other alignment in this file.

## Done when
titleView's HStack alignment is .center (or center-equivalent via alignmentGuide) instead of .firstTextBaseline; a build_run.sh screenshot taken with the supporter heart visible shows the heart.fill glyph's optical center visually level with the vertical midline of the "Basket" title text (not noticeably above or below it), across both the plain and rainbow title variants.
