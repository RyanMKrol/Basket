## Overview
Basket's pixel fonts are currently frozen at every Dynamic Type setting: Sources/Theme/Theme.swift lines 74-80 build every app font as `.custom(name, size: size * scale)` (or `.system(size:)`) with no `relativeTo:` anywhere, so a user's Larger Text setting does nothing. `Font.custom(_:size:relativeTo:)` keeps a custom typeface AND scales it with the user's setting, so the deliberate retro-pixel aesthetic and Dynamic Type scaling are NOT mutually exclusive. Both theme fonts are `.custom` (ThemePresets.swift: VT323 scale 1.15 for body, Silkscreen scale 0.78 for titles), so the fix is a direct parameter threading through the two Theme helpers that all 31 `Theme.body(...)` / `Theme.title(...)` call sites already go through. This unit does the mechanical scaling change plus an automated proof that text now scales; a separate follow-up unit re-enables the `.dynamicType` accessibility audit category and fixes layout fallout across the views.
