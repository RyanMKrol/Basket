## Overview

UI-test relaunches silently accumulate launch arguments; make each launch build its arguments fresh.

## Do

In `UITests/BasketUITestCase.swift` (~lines 32–34), `launchApp` appends to `app.launchArguments`
with `+=`. Any test that calls `launchApp` more than once on the same `XCUIApplication` accumulates
duplicate flags, and a later launch that needs *different* flags (e.g. a different `seeded:` value)
would silently keep the first launch's flags. Rebuild the argument array on every call: assign
(`app.launchArguments = [...]`) instead of appending, constructing the full array from the
`seeded`/`realTiming` parameters each time. Apply the same reset thinking to any
`launchEnvironment` keys set conditionally in that method.

## Done when

- `launchApp` produces the exact same launch arguments whether it is the 1st or 4th call on the
  same `XCUIApplication`, with no duplicates (assignment, not `+=`).
- Calling `launchApp(seeded: true)` after a previous `launchApp(seeded: false)` on the same app
  instance results in NO `-uiTestingEmpty` flag present.
- The full simulator suite passes: `xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`.
