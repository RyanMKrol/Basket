## Overview

There is no way to remove an unchecked item — the review's single biggest functional hole. Add
delete.

## Do

Add a delete affordance to unchecked (to-get) rows in `Sources/Views/ShoppingListView.swift` /
`Sources/Views/ItemRow.swift`. NOTE: the list is a `ScrollView` + `ForEach`, NOT a `List`, so
`.swipeActions` is unavailable — implement either a custom swipe-to-reveal gesture on the row or
a long-press context menu with a "Remove" action (pick whichever integrates best with the
existing check-off choreography; a context menu is the lower-risk choice). Deleting removes the
`GroceryItem` from the SwiftData context with a soft animation (`withAppAnimation`, respecting
`TestHooks`), and fires a light haptic. Give the affordance an accessibility identifier and a
VoiceOver label/hint per the house style. Update README's feature list in the same commit.
