## Overview
The add bar's keyboard-dismiss chevron (Sources/Views/AddBar.swift, accessibility id A11yID.AddBar.dismissKeyboard) currently only drops focus, leaving the typed draft text (and therefore the derived live-suggestion rows below it) on screen with no keyboard. Since the chevron is an explicit 'dismiss/cancel the input' gesture, its action should also clear the draft text, so the stale suggestion rows disappear for free through the existing text-driven derivation in ShoppingListView.
