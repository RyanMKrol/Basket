## Overview
Re-attempt of T030 (make the Home Screen widget's rows tappable to check an item off without opening the app). T030 was blocked purely on scope: its scope was [Sources/, Tests/, README.md], but the interactive rows live in BasketWidget/BasketWidgetEntryView.swift and registering the new intent needs project.yml — both out of scope, so the loop blocked on the first out-of-scope edit. The widget foundation now exists (T033, done); this widens scope to match T033's proven scope.

## Do
Add an AppIntent (usable from the widget via iOS 17+ Button(intent:)) that toggles a specific to-get GroceryItem to checked in the shared App Group store, then reloads the widget timeline. Model it on the existing Sources/Services/AddToBasketIntent.swift: put the new intent in a new file under Sources/Services/, give it a `static var containerOverride: ModelContainer?` test seam and route the reload through the existing WidgetReload.reloadTimelines indirection (do NOT call WidgetCenter directly, so it stays testable). Register the new source file in project.yml so it compiles into BOTH the app and the BasketWidget extension targets (mirror how AddToBasketIntent / the widget snapshot types are shared). Identify the target item: GroceryItem has no stable UUID (it relies on SwiftData's synthesized persistentModelID, which is not reliably reconstructable across processes from a plain string), so target the item by name (consistent with AddItem's name-based dedupe) — pass the item name as the intent parameter and, inside perform(), fetch the matching un-checked GroceryItem and check it off exactly the way the in-app check-off does (set isChecked=true and checkedAt=AppClock.now, matching ListLogic's check-off result so the item moves to 'Got it'). Keep the write safe and idempotent: if no matching un-checked item is found (already checked, or gone), no-op gracefully rather than inserting or crashing. In BasketWidget/BasketWidgetEntryView.swift, wrap each item row in a Button(intent:) bound to the new intent for the .systemMedium family (and .systemSmall where it fits), behind an `if #available(iOS 17, *)` guard if needed, falling back to the existing static row otherwise. Route any 'current time' read through AppClock.now. Note in the worklog that widget tap-to-check is not XCUITest-able and must be verified by hand on device/simulator.

## Done when
