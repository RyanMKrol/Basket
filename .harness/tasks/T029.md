## Overview

A view-only Home Screen widget (small + medium) showing what's left to get, at a glance.

## Do

Add a **WidgetKit extension target** to `project.yml` (regenerate with `xcodegen generate`; the
extension shares the App Group entitlement from T026 so it can read the store). Implement a
`TimelineProvider` that reads the **shared App Group store** and renders, in the app's Pastel Dots
look:
- **small:** the "N to get" count + the first couple of items (emoji + name);
- **medium:** the count + a longer top-items list (emoji + name), matching the in-app row style.
Reuse the pure `Emoji` service (it's plain Swift — add it to the widget target's sources) so items
show the same emoji as in-app. Handle the empty state gracefully ("All done" / basket). Keep the
timeline refresh sensible (the widget re-reads on a reasonable cadence / on app writes).

Note: the app-screenshot visual-verify hook does not capture widgets, so this task's automated bar
is compile + pure-logic tests; the actual rendering is eyeballed by hand (run the widget in the
simulator) — record that in the worklog.

## Done when

- The widget extension target builds as part of `xcodebuild` (a non-compiling widget fails CI).
- A hermetic unit test covers the **timeline entry computation** (given a set of items in a scratch
  container → the expected entry: count, the top-N items, emoji, and the empty state) — pure logic,
  no rendering.
- Full simulator suite green; README documents the widget. Manual visual check of small + medium
  rendering noted in the worklog.
