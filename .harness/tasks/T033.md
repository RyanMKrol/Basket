## Overview
Re-attempt of T029 (a view-only Home Screen widget, small + medium, reading the shared App Group store). The first attempt was blocked at rung 0 before any build/test/audit ran: the builder correctly began creating a new WidgetKit extension target's files (BasketWidget/BasketWidget.entitlements, BasketWidget/BasketWidgetExtension.swift) exactly as the spec instructed, but T029's declared scope was only Sources/, project.yml, Tests/, README.md — it never included a path for the new extension target's own directory, which is the natural/required location for any new Xcode target added via XcodeGen. This was a scope-authoring gap, not a difficulty or ambiguity problem, so the fix is adding BasketWidget/ to scope, not changing the spec's substance.

## Do
Add a WidgetKit extension target named BasketWidget to project.yml (regenerate with `xcodegen generate`), with its Swift sources living under BasketWidget/ (entitlements, extension files, TimelineProvider, views, etc. all belong there). The extension shares the App Group entitlement from T026 so it can read the shared SwiftData store. Implement a TimelineProvider that renders, in the app's Pastel Dots look: small — the "N to get" count plus the first couple of items (emoji + name); medium — the count plus a longer top-items list (emoji + name), matching the in-app row style. Reuse the pure Emoji service (plain Swift — add it to the widget target's sources) so items show the same emoji as in-app. Handle the empty state gracefully ("All done" / basket). Keep the timeline refresh cadence sensible (re-read on a reasonable interval / on app writes). Note: the app-screenshot visual-verify hook does not capture widgets, so rendering is eyeballed by hand in the simulator — record that check in the worklog.

## Done when
The BasketWidget extension target builds as part of `xcodebuild` (a non-compiling widget target fails CI). A hermetic unit test covers the timeline entry computation (given a set of items in a scratch container, produces the expected entry: count, top-N items, emoji, and the empty state) as pure logic with no rendering involved. README documents the widget. The worklog records a manual visual check of both small and medium widget rendering in the simulator.
