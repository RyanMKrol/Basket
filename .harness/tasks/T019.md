## Overview

A mistyped item can't be corrected — add rename/edit.

## Do

Add a rename affordance to to-get rows, using the same interaction family as delete (T018):
context-menu "Rename" (or the swipe-reveal's second action) that swaps the row's name label for
an inline text field (or presents a minimal alert-with-textfield — choose the approach that fits
the app's soft aesthetic and existing components best). On commit: trim whitespace, reject empty,
re-run the emoji lookup for the new name (`Emoji.forName`), update the row in place (do NOT reset
`createdAt` / list position), and refresh the quantity unit default only if the item had no
explicit quantity. Accessibility identifiers + VoiceOver labels per house style. README feature
list updated in the same commit.

## Done when

- A new XCUITest renames an item and asserts: the row shows the new name, its emoji updates to the
  new name's emoji, the list order is unchanged, and an empty rename is rejected (name unchanged).
- Existing add/check-off/quantity flows stay green; accessibility audit passes.
- Full simulator suite green; README updated.
