## Overview

A mistyped item can't be corrected. Let the user tap an item's NAME to rename it inline; renaming
re-derives the emoji and resets the quantity.

## Do

Make an item row's **name text** its own tap target that enters an inline rename. Tapping the name
label (not a swipe, not a context menu) replaces it with a prefilled `TextField` (or presents a
minimal alert-with-textfield — pick whatever fits the soft aesthetic and existing components); the
user edits the name and commits with return/submit.

Important interaction detail: today a **row tap opens the quantity editor**. The name text must
enter RENAME instead, without breaking access to the quantity editor — keep quantity reachable via
the row's `+ Qty` chip / stepper area, and make only the name label trigger rename.

On commit:
- trim whitespace; reject an empty name (keep the old name, exit rename);
- re-run `Emoji.forName` for the new name so the row's **emoji updates**;
- **reset the quantity** — clear any set amount/unit back to unset (so a renamed item starts fresh,
  since the old quantity may no longer make sense);
- keep the item's list position and `createdAt` unchanged (rename is not re-add).

Use `withAppAnimation` for the label↔field transition, give the name label and the rename field
accessibility identifiers, and add a VoiceOver label/hint ("Rename <name>"). Update the README
feature list in the same commit.

## Done when

- A new XCUITest (house rules: `accessibilityIdentifier` queries, bounded waits, screenshots) taps
  an item's name, types a new name, commits, and asserts: the row shows the new name, its emoji
  updates to the new name's emoji, the list order is unchanged, and any previously-set quantity is
  cleared.
- A second assertion: an empty rename is rejected (name unchanged).
- Tapping the `+ Qty` chip / stepper still opens the quantity editor (existing quantity flow green);
  add/check-off flows and the accessibility audit stay green.
- Full simulator suite green (`xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`); README updated.
