## Overview

Model initializer defaults use `.now`, bypassing the repo's `AppClock` test-time control.

## Do

`Sources/Models/GroceryItem.swift:28` (`createdAt: Date = .now`) and
`Sources/Models/KnownItem.swift:16` (`lastAddedAt: Date = .now`) default to the live clock,
violating the repo rule that all "current time" reads route through `AppClock.now` (see CLAUDE.md
→ Services/TestHooks). All current call sites pass `AppClock.now` explicitly, so the defaults are
an unexploded trap for future callers. Change both defaults to `AppClock.now`.

## Done when

- Both initializer defaults read `AppClock.now`; no production call site behaviour changes.
- `grep -rn '= .now' Sources/Models/` returns nothing.
- Native harness + full simulator suite pass (see README.md → Tests for both commands).
