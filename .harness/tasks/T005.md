## Overview
Tapping blank space anywhere in the app while a row's inline Quantity Editor is open currently does nothing; it should collapse the editor, the same way opening a different row's editor already collapses the previous one.

## Do
Add a tap gesture to the full-bleed `BasketBackground()` instance in `ShoppingListView.body` (~line 58, the bottom layer of the outer `ZStack`) that, when tapped, closes the currently-open editor by setting `expandedID = nil` inside a `withAnimation(.spring(response: 0.35, dampingFraction: 0.82))` block — the same animation `toggleQuantityEditor` (~line 281-300) and `clearQuantity` (~line 331-338) already use — guarded so it's a no-op when `expandedID` is already nil. Because rows' own gestures (`ItemRow`'s `.onTapGesture` on its `HStack`, plus the header and add-bar controls) sit in front of the background in the view hierarchy, SwiftUI resolves a tap to whichever gesture is deepest/most specific first, so this new background gesture will only fire for taps on genuinely blank space (below the list, the empty state, non-button header area, etc.); no change is needed in `ItemRow.swift`, and switching between rows continues to work exactly as it does today via `toggleQuantityEditor`. Do not attempt to also intercept taps on the bottom add-bar or on other rows/controls — that is out of scope for this unit; only true background taps need to close the editor.

## Done when
With an item's quantity editor open (tap a to-get row to expand it), tapping a blank area — e.g. the empty space below the last row, or the header's title/count area — collapses the editor (`expandedID` becomes nil, the stepper/unit-pills row animates closed) with no crash. Tapping a *different* row still switches the open editor to that row exactly as before (no regression to `toggleQuantityEditor`'s row-to-row behavior). Tapping blank space while no editor is open is a no-op. Verify manually via `./build_run.sh` (open an item's qty editor, tap below the list, confirm it closes) since this is gesture/UI behavior not covered by the native logic harness or XCTest.
