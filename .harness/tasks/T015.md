## Overview

No lint/format tool is configured; add SwiftLint with a light config and a CI step so the repo's
hand-maintained consistency can't drift.

## Do

Add a `.swiftlint.yml` at the repo root: start from the default rule set with `line_length`
(warning ~130), and `excluded:` covering the generated files
(`Sources/Services/EmojiTable.swift`, `Sources/Services/SuggestionDictionary.swift`), `tools/`,
`build/`, and `Tests/__Snapshots__/`. TUNE THE CONFIG (disable or relax rules) so the CURRENT
codebase passes with zero errors — do not rewrite app code to satisfy lints in this task; at most
fix trivial whitespace-class violations. Add a `Lint` job/step to `.github/workflows/ci.yml`'s
push-triggered `test` job (before the build steps): `brew install swiftlint && swiftlint --strict`
(or non-strict if warnings are intended to be advisory — pick strict-on-errors,
warnings-allowed). Document the linter in README.md.

## Done when

- `swiftlint` runs clean locally (zero errors) with the committed config, excluding the generated
  files and tools.
- CI's push-triggered job runs SwiftLint before the test steps and passes.
- README mentions the linter and how to run it; full simulator suite still green.
