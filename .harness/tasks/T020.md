## Overview

"Clear all" and delete are instant and irreversible — add a gentle undo toast.

## Do

Add an undo affordance for destructive actions: after "Clear all" in the Got-it section header
(`Sources/Views/ShoppingListView.swift`) and after a single-item delete (T018's affordance), show
a soft, theme-consistent toast ("Cleared N items — Undo" / "Removed <name> — Undo") for ~5 s
(routed through `AppClock`/`TestHooks` so tests can control it; use `withAppAnimation`). Undo
restores the deleted item(s) with their prior state (name, quantity, checked state, timestamps).
Implement restoration by deferring the actual SwiftData delete until the toast expires (soft-delete
buffer), NOT by re-inserting copies — re-inserted copies would break `persistentModelID`-based
state. The toast must not block the add bar and must be VoiceOver-announced. README updated.

## Done when

- New XCUITests: (1) Clear all → tap Undo → all got-items return with their quantities; (2) delete
  an item → Undo → item returns in its original position; (3) let the toast expire → items are
  actually gone after the TTL/expiry (bounded waits, frozen-clock friendly).
- The toast respects Reduce Motion and is announced to VoiceOver; accessibility audit green.
- Full simulator suite green; README updated.
