## Overview

"Clear all" empties the Got-it section instantly and irreversibly. Add a gentle undo toast so an
accidental tap is recoverable.

## Do

After "Clear all" in the Got-it section header (`Sources/Views/ShoppingListView.swift`) clears the
checked-off items, show a soft, theme-consistent toast ("Cleared N items — Undo") for ~5 seconds
(routed through `AppClock` / `TestHooks` so tests control the timing; animate with
`withAppAnimation`). Tapping **Undo** restores all cleared items with their prior state (name,
quantity, checked state, timestamps).

Implement restoration by **deferring the actual SwiftData delete until the toast expires**
(soft-delete buffer), NOT by re-inserting copies — re-inserted copies would break
`persistentModelID`-based state. The toast must not block the add bar and must be VoiceOver
-announced. (This task is Clear-all only; there is no single-item delete affordance in the app.)
Update the README in the same commit.

## Done when

- New XCUITests (bounded waits, frozen-clock friendly): (1) check items off → Clear all → tap Undo →
  all cleared items return with their quantities intact; (2) Clear all → let the toast expire → the
  items are actually gone afterwards.
- The toast respects Reduce Motion and is announced to VoiceOver; the accessibility audit stays
  green.
- Full simulator suite green (`xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`); README updated.
