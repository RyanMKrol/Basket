## Overview

The headline hands-free feature: an "Add <item> to my Basket" App Intent, exposed as an App
Shortcut so Siri and the Shortcuts app can add an item without opening the app.

## Do

Add an `AppIntent` (in the app target — App Shortcuts do not need a separate extension) that takes
an item name parameter and inserts a new `GroceryItem` into the **shared App Group store** (from
T026), deriving the emoji via `Emoji.forName` exactly like an in-app add, and recording it in the
suggestion memory (`KnownItems.rememberAdd`) so it behaves identically to typing it. It must be
duplicate-aware in the same spirit as the in-app add (re-adding an existing item bumps it rather
than duplicating). Register it as an `AppShortcutsProvider` phrase ("Add \(item) to Basket") so
"Hey Siri, add milk to my Basket" works. Route any time reads through `AppClock.now`. Update the
README feature list.

## Done when

- A hermetic unit test drives the intent's `perform()` against a **scratch/in-memory** model
  container (never the real store) and asserts: a new item with the right name + derived emoji is
  inserted; re-invoking with an existing name bumps rather than duplicates; the suggestion memory
  records it.
- The app builds with the App Shortcut registered (the intent + `AppShortcutsProvider` compile into
  the app target); full simulator suite green (`xcodegen generate && xcodebuild test -project
  Basket.xcodeproj -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`).
- README documents the Siri phrase. (Actual Siri invocation is verified by hand on a device — note
  it in the worklog; it is not part of the automated bar.)
