## Overview
Accessibility Nutrition Labels are the App Store Connect metadata surface where an app declares which accessibility features it supports (VoiceOver, Larger Text, Reduced Motion, etc.). Basket has audit-tested VoiceOver support today and, once the two code-side units in this cluster land, honest Larger Text and Reduced Motion stories, but nothing is declared in ASC. This is store-visibility metadata edited in the owner's App Store Connect account, not code: gate needs-human. The loop must prepare nothing further, record failed:blocked if it picks this up, and never auto-complete it.

## Do
## Do (owner checklist, in App Store Connect)

1. Sign in to App Store Connect > My Apps > Basket, and open the Accessibility (Nutrition Labels) section of the app's App Store metadata.
2. Declare **VoiceOver: supported**. Evidence: UITests/AccessibilityAuditTests.swift runs performAccessibilityAudit over the main list, empty state, quantity editor, clear-all undo toast, and about sheet with the label, trait, element-detection, and description categories enforced in CI; Sources/Views carries curated accessibility labels, hints, actions, and identifiers throughout (rows, check circles, rename, quantity editor). Spot-check once with VoiceOver on a device before declaring.
3. Declare **Larger Text: supported**. Evidence: Theme fonts scale via Font.custom relativeTo, a UI test asserts text grows under an accessibility content size, and the .dynamicType audit category passes in CI. Only declare AFTER the Dynamic Type units are merged. If those units capped scaling below the maximum accessibility sizes, check Apple's current Nutrition Label guidance on partial or capped support and declare only what is honest.
4. Declare **Reduced Motion: supported**. Evidence: ClearedCelebration, LaunchFlourish, ShoppingListView transitions, and (after the sibling unit) the per-item SparkleBurst all suppress or calm motion under the system setting. Only declare after that unit is merged.
5. Do NOT declare **Sufficient Contrast** (the soft pastel palette intentionally fails contrast heuristics; the audit's .contrast category is excluded by design pending a product decision) and do NOT declare **Dark Interface** (the app ships a single light theme). Captions and Audio Descriptions are not applicable (no audio or video content).
6. Optionally evaluate **Differentiated Without Color Alone**: checked state is conveyed by a checkmark glyph, strikethrough, and section move, not colour alone; declare it if a quick manual pass against Apple's wording confirms.
7. Save the declarations (they publish with the next app version's metadata). Add a short note to README.md recording which labels were declared and the evidence for each. No em dashes in any ASC copy or README text.
