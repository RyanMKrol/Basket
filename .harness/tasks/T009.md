## Overview

Short keywords match by bare word-prefix, so "Toilet paper" → 🥣 (via `pap`) and "Carpet cleaner"
→ 🐠 (via `carp`). Require exact-word (or plural) matches for short keywords.

## Do

In the match logic emitted by `tools/gen_emoji.py` (the simple-keyword word-prefix pass in the
generated `EmojiTable.swift`), keywords of length ≤ 4 currently match ANY word starting with them.
Change the emitted matcher so keywords with length ≤ 4 require the word to be an EXACT match or a
simple plural (`word == kw || word == kw + "s" || word == kw + "es"`), while keywords of length
≥ 5 keep the existing `hasPrefix` behaviour (which powers the deliberate truncated stems like
`strawberr`, `anchov`, `pastr`). Regenerate the table via the generator — no hand edits.
This kills the verified false-positive class: "toilet paper"/"paper towels" → 🥣 (via `pap`),
"carpet cleaner" → 🐠 (via `carp`), "lingonberry jam" → 🐟 (via `ling`), "hamper" → 🍖 (via `ham`),
"tikka masala paste" → 🌽 (via `masa`).

## Done when

- Through the real cascade: "toilet paper" and "paper towels" NO LONGER match via `pap`;
  "carpet cleaner" no longer matches `carp`; "hamper" no longer matches `ham`; while "pap",
  "ham", "hams", "carp" alone still map to their correct emoji.
- Existing short-keyword plurals still work ("figs", "peas", "yams" resolve as before) and the
  ≥ 5-char prefix stems ("strawberries" via `strawberr`) are unaffected.
- Native harness gains checks covering at least 4 of the false-positive fixes above; coverage
  audit stays at 0 fall-throughs; full simulator suite passes.
