## Overview

`Measure.parse` accepts "-5" as 5 and "2x500" as 2500, contradicting its own doc comment.

## Do

`Sources/Services/Measure.swift` — `parse(_:unit:)` (~line 143) filters the input to digits and
"." before parsing, so a leading minus is dropped ("-5" → 5, accepted, though the doc comment at
~line 141 says zero-or-negative is rejected) and digits concatenate across junk ("2x500" → 2500).
Unreachable from the app's number pads (no minus key) but reachable via hardware keyboard. Fix:
reject any input whose first non-whitespace character is "-", and parse only the FIRST contiguous
number group (digits with an optional single decimal separator, comma or dot) instead of
concatenating all digits — "2x500" parses as 2, "750 ml" still parses as 750, "1,5" still parses
as 1.5.

## Done when

- New XCTest cases in `Tests/MeasureTests.swift` assert: "-5" → nil, "2x500" → 2, "750 ml" → 750,
  "1,5" → 1.5, ".5" behaviour documented by a test, "" → nil.
- The forgiving behaviours documented in README (ignores unit letters, comma decimals, rounds
  counts) still hold; native harness Measure checks still pass.

  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`).
