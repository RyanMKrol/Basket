## Overview

The coverage audit only checks that SOMETHING matched, not that it's right — add a golden subset
with expected emoji so semantic regressions become visible.

## Do

Extend `tools/audit_coverage.swift` with a CORRECTNESS mode: a curated golden table of ~150–200
`item → expected emoji` pairs spanning every corpus category (food staples, UK/US synonym pairs,
household, toiletries, pharmacy, baby, pet, brands, and the fixed regressions from the earlier
tasks: marmalade → 🍊, toilet paper → 🧻, ginger beer head-noun case, rice cakes → 🍘, chicken
nuggets → 🍗). The audit runs each golden item through the real cascade and FAILS (non-zero exit)
on any mismatch, printing item / got / expected. Keep the existing coverage counting unchanged.
Document how to run it in README.md → Tests alongside the existing audit instructions.

## Done when

- `tools/audit_coverage.swift` compiled against the generated table exits non-zero when a golden
  expectation mismatches (verify by temporarily flipping one expectation), zero on the committed
  golden set.
- The golden set passes against the current table, includes ≥ 150 entries, and covers at least 8
  distinct categories.
- README documents the correctness audit; native harness + full simulator suite pass.
