## Overview
Typing "salad" into Basket's add bar currently shows a canned-goods emoji (🥫) instead of a salad-appropriate one, because the curated EmojiTable has no bare-word entry for "salad" (only the multi-word "salad cream" -> 🥫, which can't match a shorter input), so the lookup falls through to the semantic (NLEmbedding) fallback, which lands nearest a condiment/canned-goods anchor. This task fixes it by adding "salad" as its own curated keyword so the fast, exact first stage resolves it correctly and the semantic fallback never runs for this word.

## Do
In tools/gen_emoji.py, add a new single-word entry `("salad", "🥗")` (the existing salad-bowl emoji, U+1F957, already used elsewhere in the table e.g. for "banchan", "kachumbari", "causa") to the RAW data string, in the produce/prepared-dish section. Add it to gen_emoji.py's RAW, not tools/emoji_supplement.txt — curated RAW entries are appended first and win on any keyword collision, so RAW is the correct home for a common single-word grocery item like this (this specific entry has no collision either way, but keep it with the other curated data for consistency). Do NOT hand-edit Sources/Services/EmojiTable.swift directly — run `python3 tools/gen_emoji.py Sources/Services/EmojiTable.swift` to regenerate it per the repo's emoji pipeline (see CLAUDE.md "Emoji pipeline" section). Also add a golden-subset regression pin `("salad", "🥗")` to the `golden` array in tools/audit_coverage.swift (near the other food-staple entries) so this exact mapping is guarded against future regression.
