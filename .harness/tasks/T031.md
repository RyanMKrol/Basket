## Overview

A quick-add "+" affordance on the widgets: a button that jumps into the app with the keyboard up
(via the T028 deep link) — plus a dedicated small "add" widget for one-tap access.

## Do

Building on the deep link (T028) and the widget target (T029):
1. Add a **"+" add button** to the widget(s) that opens the app via the `basket://add` deep link
   (a `widgetURL(...)` / `Link` to that URL) so tapping it lands in the app with the add bar
   focused and the keyboard up. (You cannot type into the widget itself — this is the intended
   "take me into the app to add" behaviour.)
2. Provide the widget in the variants the owner wants: a **view-only** list widget (T029), and an
   **add** widget (a compact small widget that is mostly the "+" quick-add button), and — where it
   fits the medium layout — a **combined** widget that shows the list AND carries the "+" button.
   Use a `WidgetBundle` / configuration so the user can choose which to place.
Match the Pastel Dots look; handle small + medium sizes.

Note: widget rendering isn't XCUITest-able; the deep-link landing (keyboard up) IS covered by
T028's app-side test. This task's automated bar is compile + that the widget URL resolves to the
add deep link; rendering is eyeballed by hand.

## Done when

- The widget bundle builds with the view-only, add, and combined variants selectable; the "+"
  button's `widgetURL` is the `basket://add` deep link (assert the URL wiring in a small unit test
  if practical).
- Tapping "+" in the simulator opens the app with the keyboard up (this leans on T028, which is
  automated); manual visual verification of the three variants noted in the worklog.
- Full simulator suite green; README documents the widget variants.
