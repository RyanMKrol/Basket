## Overview

Basket ships one look (Pastel Dots) but carries ~16 other themes reachable only via a launch env
var. Remove them: hardcode the Pastel Dots look and delete every other theme and its plumbing. This
also eliminates the `Theme.current` mutable-global concurrency risk (it becomes a constant).

## Do

The app should have exactly ONE look — the current default, **Pastel Dots** — with no runtime theme
selection at all:

- **`Sources/Theme/`** — keep the `ThemeStyle` shape and the Pastel Dots values, delete every other
  preset. In `Theme.swift`: remove `Theme.all`, `Theme.select(id:)`, and the `static var current`
  mutable global; make the look a single hardcoded constant (a `let`, or fold the Pastel Dots values
  directly into the `Theme.paper` / `Theme.leaf` / … live accessors). In `ThemePresets.swift`:
  delete the `.soft` / `.pixelPantry` / `.deepDive` / `.cozyCabin` / `.nightArcade` presets and the
  `ThemeStyle.fresh` array; keep only what Pastel Dots needs (if nothing else remains, delete the
  file and regenerate the project with `xcodegen generate`).
- **`Sources/BasketApp.swift`** — remove the `BASKET_THEME` env-var read and the `Theme.select(...)`
  call (lines ~13–14). `preferredColorScheme(Theme.current.isDark ? .dark : .light)` (line ~48)
  becomes a plain `.light` (Pastel Dots is a light look).
- **`UITests/AccessibilityAuditTests.swift`** — `testMainListPassesAuditInEveryTheme` iterates the
  themes; collapse it to a single-look audit of the main list (no theme switching), keeping the
  audit assertions.
- **`README.md`** — remove the "theme / `BASKET_THEME` env var / other themes ship in code"
  documentation; state plainly that the look is Pastel Dots.

The rendered result must be **visually identical to today's default** — this is a removal of dead
alternatives, not a restyle.

## Done when

- No `Theme.current` mutable global, no `Theme.all`, no `Theme.select`, and no `BASKET_THEME` read
  remain anywhere (`grep -rn 'BASKET_THEME\|Theme.select\|Theme.all\|var current' Sources/` returns
  nothing); the app compiles with the Pastel Dots values as the single hardcoded look.
- The accessibility audit test no longer loops over themes and still passes.
- The app looks unchanged from the current default (verify via `build_run.sh` screenshot); README no
  longer mentions other themes or the env var.
- Full simulator suite green: `xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`.
