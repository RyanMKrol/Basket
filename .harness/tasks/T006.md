## Overview

`Theme.current` is unprotected global mutable state — a Swift 6 strict-concurrency error in waiting.

## Do

`Sources/Theme/Theme.swift:61` declares `static var current: ThemeStyle = .pastelDots` — written
once in `BasketApp.init`, read from the main thread, but formally a data race and a hard error
under Swift 6 strict concurrency. Fix it the same way the repo already handled this pattern in
`Sources/Views/LaunchFlourish.swift:9`: either annotate the enclosing type/property `@MainActor`,
or mark it `nonisolated(unsafe)` with a short why-comment matching the `LaunchOnce` precedent.
Prefer `@MainActor` if it compiles cleanly everywhere `Theme.current` is referenced (all reads are
from SwiftUI views); fall back to `nonisolated(unsafe)` + comment otherwise. Do not change how or
when the theme is selected.

## Done when

- `Theme.current` carries either `@MainActor` isolation or `nonisolated(unsafe)` with a
  rationale comment; no callers changed behaviourally.
- The app builds with zero new warnings and the full simulator suite passes
  (`xcodegen generate && xcodebuild test -project Basket.xcodeproj -scheme Basket -destination
  'platform=iOS Simulator,name=iPhone 17 Pro'`).
