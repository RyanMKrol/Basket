## Overview
Axiom's test-debugger discipline: when a UI test fails to find an element, screenshot the live state FIRST - if the element is genuinely absent from the screen, it is an APP bug; never 'fix' the test by loosening or rewriting selectors until it passes. Basket's audit guidance judges rendered pixels but has no rule catching an unattended builder who responds to a failing accessibility-identifier query by rewriting the query (or renaming the identifier) to make it pass, masking a real regression - a plausible cheap-model failure mode. Edits .harness/custom/ overlay files, so gate: needs-human; ordered after the other visual-verify-audit.md edits to avoid same-file collisions.
