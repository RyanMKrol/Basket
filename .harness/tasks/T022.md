## Overview

No way to get the list out of the app — add share-sheet export as plain text.

## Do

Add a share affordance (a small share icon in the header near the ⓘ, matching the soft style) that
exports the CURRENT to-get list via the system share sheet (`ShareLink` or
`UIActivityViewController` wrapped for SwiftUI) as plain text, one item per line in the format
"<emoji> <Name>" plus quantity when set ("🍅 Tomatoes ×2", "🥛 Milk 500 ml" — reuse the existing
row formatting helpers in `Sources/Services/Formatting.swift`/`Measure` rather than reformatting).
Checked-off items are excluded. The button is hidden (or disabled) when the list is empty.
Accessibility identifier + VoiceOver label per house style. No new privacy-manifest entry is
needed (no required-reason API), but confirm and note it in the worklog. README updated.

## Done when

- A unit test pins the export string format for a list containing: a plain item, an item with a
  count quantity, and an item with an ml/L quantity (exact expected multi-line string).
- A new XCUITest asserts the share button exists with items present and is absent/disabled on an
  empty list (do NOT attempt to assert inside the system share sheet).
- Full simulator suite green; README updated.
