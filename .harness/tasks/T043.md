## Overview
Basket has ZERO performance coverage today: no measure{} blocks, no XCTMetric, no baseline anywhere in Tests/ or UITests/ (verified by grep). A perf regression ŌĆö e.g. the emoji cascade or suggestion scoring getting slow ŌĆö would ship silently. The Axiom plugin ships `xcprof`, a headless Instruments front-end: `xcprof record` captures a .trace via xctrace with no Instruments GUI, `xcprof analyze --json` turns it into structured CPU hot-frame JSON, and `xcprof compare <baseline.trace> <current.trace> --fail-on-regression` exits 3 when any function's inclusive CPU-cycle share rises by >= --threshold-pct (default 5 percentage points). This task adds `tools/profile_app.sh` ŌĆö a script that records a CPU trace around one fixed, deterministic XCUITest workload and analyzes it ŌĆö plus a committed baseline analyze-JSON snapshot and a documented local A/B compare workflow for perf-sensitive changes.

HONESTY CAVEAT (from the plugin's own xcprof reference, and it MUST be carried into the README docs this task writes ŌĆö do not oversell): this is CPU-only profiling. Memory and macOS energy report `not_exportable`: "Allocations/Leaks data lives in the trace's event store (no XML table ŌĆö open it in Instruments.app)". This tooling does NOT do leak detection or memory-growth gating, and the docs must say so explicitly.

SECOND HONESTY POINT the docs must reflect: `xcprof compare` diffs two .trace bundles, NOT two JSON files. The committed baseline JSON is a human-readable reference snapshot of the hot frames (what 'normal' looks like), not a compare input. Cross-machine trace comparison is unreliable anyway (different hardware shifts cycle shares), so the regression workflow is a LOCAL two-trace A/B: record a baseline trace on main, record a current trace on your branch, compare the two on the same machine. CI gating is explicitly OUT of scope for this task (GitHub runners don't have the Axiom plugin, and cross-machine baselines are noise) ŌĆö the README must state that CI gating is deferred to a possible follow-up.
