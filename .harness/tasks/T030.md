## Overview

Make the widget interactive: tap an item in the widget to check it off, without opening the app
(iOS 17+ interactive widgets).

## Do

Add an `AppIntent` (usable from the widget, iOS 17+ `Button(intent:)`) that toggles a specific
`GroceryItem` to checked in the **shared App Group store** and reloads the widget timeline
(`WidgetCenter.reloadTimelines`). Wire each item row in the medium (and where it fits, small)
widget from T029 to that intent so a tap checks the item off, mirroring the in-app check-off result
(the item moves to "Got it"). Keep the shared-store write safe and idempotent. Route time reads
through `AppClock.now`.

Note: as with T029, widget rendering/interaction isn't XCUITest-able — the automated bar is compile
+ a hermetic test on the toggle intent; the real tap-to-check is verified by hand.

## Done when

- A hermetic unit test drives the check-off intent's `perform()` against a scratch container and
  asserts the target item becomes checked (and an already-checked item is handled sanely).
- The widget target still builds; the interactive rows compile against the iOS 17+ API with an
  availability guard if needed. Full simulator suite green; README updated. Manual check-off-from-
  widget verification noted in the worklog.
