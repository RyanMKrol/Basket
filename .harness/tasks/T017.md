## Overview

Accessibility identifiers are duplicated as string literals between app views and UI tests; share
them via one compiled-into-both constants file, like `SharedFixtures` already does for seed names.

## Do

Create a shared `enum A11yID` (e.g. in `Sources/Support/A11yIDs.swift`) declaring every
accessibility identifier currently duplicated as literals: the add bar (`addBar.textField`, add
button, dismiss, suggestion chips), item rows (`itemRow.check.<name>` — provide a
`static func check(_ name: String) -> String` style helper), quantity editor
(`quantityEditor.increase` etc.), header/got-section ids in `ShoppingListView`. Wire the file into
BOTH targets the same way `Sources/Support/SharedFixtures.swift` is (see `project.yml` ~lines
96–97 for the mechanism). Replace the literals in `Sources/Views/*.swift` AND in every
`UITests/*.swift` with the shared constants. Also fold in the duplicated "N to get" header-count
formatter (built independently at `ShoppingListView.swift:185` and
`UITests/BasketUITestCase.swift:108`) so a copy change can't silently break the tests.

## Done when

- `grep -rn '"itemRow.check\|"addBar\.\|"quantityEditor\.' Sources/ UITests/` finds only the
  shared constants file (no scattered literals).
- The header-count string is built by one shared function used by both the view and the test base
  class.
- Full simulator suite green (`xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`) — the UI tests
  passing proves the identifiers still line up.
