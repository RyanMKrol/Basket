## Overview
tools/export_ui_screenshots.sh always exports EVERY XCTAttachment from the UI-test run (slow, noisy), and when a UI test fails because the app itself crashed, nothing surfaces the crash reason; the caller just sees a timeout or element-not-found. Add (1) a --failures-only mode that exports only failure-associated attachments, and (2) an automatic post-run crash triage step that sweeps fresh .ips crash reports and summarizes them with the Axiom plugin's xcsym binary when available, degrading gracefully to just listing the .ips paths when it is not.

### 2. Do not fail-fast on test failure any more

Currently a failed test run exits 1 before exporting anything. Change to: record the xcodebuild exit status (e.g. `TEST_FAILED=1` in the `if !` branch, after printing the log tail), then STILL run the attachment export (failure screenshots are exactly what is needed then) and the crash triage below, and finally `exit 1` at the end of the script if the run failed. A passing run exits 0. Note `set -e` applies; keep every may-fail command inside an `if`/`|| true` shape deliberately, never accidentally.

### 3. Crash triage sweep (runs only when the test run failed)

- Before the xcodebuild invocation, create a timestamp marker: `RUN_MARKER="$(mktemp)"`.
- After a FAILED run, sweep for fresh crash reports from the app:

```sh
CRASHES=$(find "$HOME/Library/Logs/DiagnosticReports" -name 'Basket*.ips' -newer "$RUN_MARKER" 2>/dev/null || true)
```

- If none are found, print a one-liner saying no app crash reports were produced by this run.
- If found, locate xcsym. Honor an env override first, then discover the newest Axiom plugin install:

```sh
XCSYM="${XCSYM:-$(ls -d "$HOME"/.claude/plugins/cache/axiom-marketplace/axiom/*/bin/xcsym 2>/dev/null | sort -V | tail -1)}"
```

(Current concrete path on this machine: /Users/ryankrol/.claude/plugins/cache/axiom-marketplace/axiom/27.0.0-beta.22/bin/xcsym, but never hardcode the version; use the discovery above.)
- If `XCSYM` is empty or not executable, DEGRADE GRACEFULLY: print the .ips file paths with a note that the Axiom plugin's xcsym was not found so the reports are unsymbolicated, and continue. The triage step must NEVER make the script fail on its own; the script's exit code reflects only the test run result.
- If xcsym is present, for each .ips run `"$XCSYM" crash "$file" --format=summary || true` and print the output under a clear heading. (If that flag name differs on the installed version, check `"$XCSYM" crash --help` and use the summary-style output it offers.)
