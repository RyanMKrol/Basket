## Overview
When BasketTests runs, the host app launches WITHOUT -uiTesting, so BasketApp.init opens the REAL App Group SwiftData store (AppGroup.storeURL), seeds starter items into it, and can even fatalError if that container fails — every unit-test invocation (including every harness LOCAL_DOD run) pays for and MUTATES production-shaped state. Add an XCTest-host detection hook to TestHooks (the existing launch-arg/env determinism switchboard — house convention, no ad-hoc globals) and use it in BasketApp.init to swap in an inert startup: a fresh in-memory store and no seeding. The unit tests themselves already build their own in-memory containers (Tests/ModelTests.swift makeContainer()), so nothing in Tests/ depends on the host's store.
