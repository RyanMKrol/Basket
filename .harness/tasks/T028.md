## Overview

Groundwork for the widget's "+" button and a nice quick-add: a deep link that opens the app with
the add bar already focused and the keyboard up, ready to type.

## Do

Register a URL scheme (e.g. `basket://add`) in `project.yml` (regenerate) and handle it in the app
(`.onOpenURL` on the root scene / `BasketApp`) so that launching or foregrounding the app via that
URL sets the existing `@FocusState addBarFocused` (in `ShoppingListView`) to `true`, raising the
keyboard on the add bar ready for input. This is purely app-side (no App Group needed) and is the
target the widget's add button (T031) will deep-link to. Keep it robust to the app being cold-
launched vs already running. Give the flow whatever accessibility affordance is appropriate; update
the README.

## Done when

- A new XCUITest launches the app with the `basket://add` URL (via
  `XCUIApplication().launch()` with the URL, or `openURL`) and asserts — using bounded waits and
  `accessibilityIdentifier` — that the add-bar text field is focused / the keyboard is up, for both
  a cold launch and an already-running app.
- Normal launches (no URL) are unaffected: the keyboard does NOT auto-raise (existing tests green).
- Full simulator suite green (`xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`); README updated.
