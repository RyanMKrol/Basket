## Overview
Migrate the app to Swift 6 language mode (strict concurrency) while the violation surface is still tiny. project.yml:20 currently pins SWIFT_VERSION "5.0" with no strict-concurrency flag. The known violation surface (verified by compiling under -swift-version 6): LaunchFlourish.swift:9 `nonisolated(unsafe) static var fired` (LaunchOnce), AddToBasketIntent.swift:64 `static var containerOverride: ModelContainer?` (mutable unisolated global — guaranteed Swift 6 error), the other AppIntents `static var`s in that file (typeDisplayRepresentation, defaultQuery, title, description — mutable statics, should be `let` or computed), Emoji.swift:17 `static let cache = NSCache<...>` (non-Sendable type, confirmed error under Swift 6), SemanticEmoji.swift:22 `static let embedding = NLEmbedding...` (same, confirmed), and Haptics.swift's UIFeedbackGenerator statics (MainActor-isolated UIKit types). TipJar is already @MainActor; TestHooks/AppClock/Theme statics are immutable Sendable values and are fine. This is behavior-preserving: no user-visible change, the full existing suite is the gate. SEQUENCING PREFERENCE: this should ideally land BEFORE T033 (view-only widget), because T033 adds a widget extension — a second process compiling shared Sources/ files — which is exactly when latent concurrency assumptions start biting; the coordinator/owner may want to order this task ahead of T033 in the backlog (do NOT express this as a dependsOn — T033 must not wait on this task formally).
