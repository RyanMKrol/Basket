## Overview
build_run.sh launches the app and takes a screenshot but discards the app's console output entirely. The app logs through os.Logger in three places (Sources/BasketApp.swift and Sources/Services/KnownItems.swift, subsystem com.ryankrol.basket category "persistence"; Sources/Services/TipJar.swift category "TipJar"), so a failed SwiftData save during a visual verification is invisible: the screenshot looks fine while persistence silently failed. Add a best-effort console error check to the end of build_run.sh using the Axiom plugin's xclog binary, degrading to a skip note when xclog is absent.

## Do

All changes are in `build_run.sh` (bash, `set -euo pipefail`; it resolves the simulator to a UDID in `$SIM`, launches bundle id com.ryankrol.basket via `simctl launch`, sleeps 3s, then saves screenshots/latest.png).

After the screenshot step, add a console error check:

1. Locate xclog, env override first, then newest Axiom plugin install:

```sh
XCLOG="${XCLOG:-$(ls -d "$HOME"/.claude/plugins/cache/axiom-marketplace/axiom/*/bin/xclog 2>/dev/null | sort -V | tail -1)}"
```

(Current concrete path on this machine: /Users/ryankrol/.claude/plugins/cache/axiom-marketplace/axiom/27.0.0-beta.22/bin/xclog; never hardcode the version.)

2. If `XCLOG` is empty or not executable, print a skip note (e.g. "xclog not found, skipping console error check") and finish normally. The whole check is best-effort: it must NEVER change the script's exit code or abort the run, so wrap every xclog invocation with `|| true` and mind `set -euo pipefail`.

3. If present, use xclog's post-mortem `show` mode, which reads os_log/Logger output already emitted by the just-launched app (launch mode would relaunch the app itself, which we do not want since simctl launch already ran):

```sh
CONSOLE_LOG="$SHOT_DIR/console-errors.log"
"$XCLOG" show "$APP_NAME" --device "$SIM" --last 2m --filter '(?i)error|fault' > "$CONSOLE_LOG" 2>/dev/null || true
```

Notes: `show` mode captures NSLog/os_log/Logger (which covers all three Logger call sites; the app does not rely on print for errors); `--device` takes the simulator UDID which `$SIM` already holds when resolution succeeded. The 2 minute window plus the process-name scoping keeps output limited to this launch in practice.

4. If the captured file is non-empty, print a clearly visible warning heading (e.g. "Console errors detected during launch:") followed by its contents, and KEEP the file next to the screenshot so a harness worklog can attach it. If empty, delete the file and print a clean one-liner (e.g. "No console errors during launch"). Either way exit 0; this step surfaces problems, it does not gate.

Add one or two sentences: after launch the script also checks the app's console for error/fault log lines via the Axiom plugin's xclog when installed, writing any findings to screenshots/console-errors.log, and skips the check quietly when xclog is absent.
