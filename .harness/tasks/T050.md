## Overview
WCAG 2.3.3 (Animation from Interactions): decorative motion should be suppressed when the user enables Reduce Motion. Sources/Views/ItemRow.swift line 102 plays SparkleBurst() on EVERY check-off, unconditionally: `.overlay { if isChecking { SparkleBurst() } }`. This is the app's single most frequent animation, yet the codebase's two rarer bursts are already gated correctly: Sources/Views/ClearedCelebration.swift line 14 and Sources/Views/LaunchFlourish.swift line 31 both wrap their SparkleBurst in `if !reduceMotion`. So this is an inconsistency with an established in-repo convention, not a new convention. Note that TestHooks' withAppAnimation / .unlessUITesting only disable animation under UI testing, not under the accessibility setting, so nothing else is covering this.
