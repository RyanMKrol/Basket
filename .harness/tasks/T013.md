## Overview

`Measure`, `Seasonality`, and `Formatting` have no XCTest coverage — all their checks live only in
the native harness (`tools/main.swift`), invisible to the simulator suite's coverage data.

## Do

Port the `Measure`, `Seasonality`, and `Formatting` checks from `tools/main.swift` (~lines
127–184 for Measure) into proper XCTest classes under `Tests/` (e.g. `Tests/MeasureTests.swift`,
`Tests/SeasonalityTests.swift`, `Tests/FormattingTests.swift`), following the style of the
existing `Tests/BasketTests.swift`. While porting, ALSO cover the gaps the harness misses:
`Measure.changeUnit` g↔kg (only ml↔L is checked today), `Measure.step` behaviour around the 1000
threshold, and `Seasonality.holidayAccent` boundary days (Oct 23/24, Dec 26/27, Jan 1/2) using
fixed dates. Leave `tools/main.swift` untouched in this task (dedupe is a later decision) —
this task only ADDS simulator-side coverage.

## Done when

- Three new test files exist under `Tests/` and run in the `Basket` scheme's test action
  (xcodegen picks up `Tests/` automatically — verify with `xcodebuild test`).
- g↔kg conversion, the 1000-threshold stepping, and holiday boundary days are asserted with fixed
  deterministic dates (no live clock).
- Full simulator suite green: `xcodegen generate && xcodebuild test -project Basket.xcodeproj
  -scheme Basket -destination 'platform=iOS Simulator,name=iPhone 17 Pro'`; native harness still
  green.
